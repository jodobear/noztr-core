8. `handoff.md`

## Core Rules

- Respect canonical artifact precedence.
- Work one phase at a time.
- Use one prompt per phase.
- Record tradeoffs for material decisions.
- Do not close a phase while high-impact ambiguity remains `decision-needed`.
- Ask clarifying questions before costly or irreversible work.
- Run declared quality gates after code changes.
- Update `handoff.md` after meaningful progress.

## Session Completion

- Run required quality gates.
- Update the decision log if defaults changed.
- Update `handoff.md`.
- Follow the project profile's local and remote landing workflow.
