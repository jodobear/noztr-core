# noztr-core 0.1.0-rc.4

Date: 2026-03-23
Release type: breaking rc

## Summary

`0.1.0-rc.4` is the fourth public release candidate for `noztr-core`.

This RC continues the pre-`1.0` public-surface cleanup: grouped routes now carry more of the
context, route-local read models are shorter and easier to scan, and the current migration path is
tighter for both humans and LLMs. Wire behavior, ownership posture, and the kernel-vs-SDK boundary
stay unchanged.

## Public Highlights

- renamed remaining high-confidence route-local read models and simple role types
- kept the current downstream upgrade path consolidated in one migration guide
- widened root contract-smoke coverage for the newly canonical public names
- tightened one `nip04` negative-path test so it exercises the intended ciphertext-length failure

## Breaking Changes

- renamed route-local public types across:
  - `nip04`
  - `nip10_threads`
  - `nip22_comments`
  - `nip29_relay_groups`
  - `nip30_custom_emoji`
  - `nip32_labeling`
  - `nip72_moderated_communities`
  - `nip98_http_auth`
  - `nipb0_web_bookmarking`
  - `nipb7_blossom_servers`
  - `nipc0_code_snippets`
- no wire formats, parser contracts, or ownership rules changed
- migration guide:
  - [../guides/migrating-from-0.1.0-rc.3.md](../guides/migrating-from-0.1.0-rc.3.md)

## Compatibility Notes

- Zig toolchain floor for this RC line remains `0.15.2`
- optional I6 exports remain build-flag gated
- deterministic crypto behavior and protocol scope are unchanged by this RC

## Verification

- `zig build lint`
- `zig build test --summary all`
- `zig build`
- `zig build release-check`

## Upgrade Guidance

- if you depend on `noztr-core`, review [../guides/migrating-from-0.1.0-rc.3.md](../guides/migrating-from-0.1.0-rc.3.md)
- update direct public type references to the shorter route-local names
- refresh local symbol indexes or LLM context packs so they reflect the canonical names
